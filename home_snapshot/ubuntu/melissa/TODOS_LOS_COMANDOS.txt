════════════════════════════════════════════════════════════════════════════════
                                GRUPO 1: RÁPIDOS
                          (Sin parámetros - Solo escribir)
════════════════════════════════════════════════════════════════════════════════

1. /status
   ESCRIBE: /status
   QUÉ PASA: Ver estado de TODAS las instancias
   MUESTRA: Online/Total, estado individual, latencia de cada una
   USES: Mañana al abrir, para orientarte

2. /alerts
   ESCRIBE: /alerts
   QUÉ PASA: Ver alertas pendientes (no reconocidas)
   MUESTRA: Lista de alertas con timestamp, tipo, instancia
   USES: Para ver si hay problemas

3. /ack
   ESCRIBE: /ack
   QUÉ PASA: Reconocer TODAS las alertas pendientes
   MUESTRA: Confirmación de cuántas se reconocieron
   USES: Después de leer las alertas

4. /help
   ESCRIBE: /help
   QUÉ PASA: Mostrar lista de comandos
   MUESTRA: Todos los comandos con descripción
   USES: Si olvidas qué comando usar

5. /conversations
   ESCRIBE: /conversations
   QUÉ PASA: Ver chats activos de TODAS las instancias
   MUESTRA: Top 10 chats, nombre, ID, último mensaje, hace cuánto
   USES: Ver clientes nuevos, monitoreo de chats

6. /chats
   ESCRIBE: /chats
   QUÉ PASA: Alias para /conversations (lo mismo)
   MUESTRA: Lo mismo que /conversations
   USES: Si prefieres escribir /chats

7. /list
   ESCRIBE: /list
   QUÉ PASA: Re-listar conversaciones
   MUESTRA: Lo mismo que /conversations nuevamente
   USES: Si quieres volver a ver la lista de chats

════════════════════════════════════════════════════════════════════════════════
                          GRUPO 2: CON PARÁMETROS
                      (Requieren información adicional)
════════════════════════════════════════════════════════════════════════════════

8. /analyze [instancia]
   ESCRIBE: /analyze clinica-bella
   ESCRIBE: /analyze restaurante-central
   QUÉ PASA: Análisis PROFUNDO y MEGA inteligente de una instancia
   MUESTRA:
   - Satisfacción de clientes (%)
   - Temas trending
   - Anomalías detectadas
   - Correlaciones históricas
   - Acciones recomendadas
   - Predicciones
   USES: Cuando quieres entender una instancia en profundidad

9. /what-happened [instancia] [horas]
   ESCRIBE: /what-happened clinica-bella 6
   ESCRIBE: /what-happened restaurante-central 24
   ESCRIBE: /what-happened hotel 12
   QUÉ PASA: Ver histórico de eventos en últimas N horas
   PARÁMETROS:
   - instancia: nombre de la instancia (OBLIGATORIO)
   - horas: número de horas atrás (OPCIONAL, default 6)
   MUESTRA:
   - Eventos cronológicos
   - Cambios de estado
   - Alertas que pasaron
   - Causalidad entre eventos
   USES: Investigar qué pasó ayer/hace poco

10. /root-cause [evento]
    ESCRIBE: /root-cause latency_spike
    ESCRIBE: /root-cause offline_restaurant
    ESCRIBE: /root-cause high_error_rate
    QUÉ PASA: Análisis de CAUSA RAÍZ - por qué pasó algo
    MUESTRA:
    - Qué causó el problema
    - Cuándo pasó algo similar antes
    - Cómo se resolvió la última vez
    - Probabilidad de la causa
    USES: Cuando algo se rompe y necesitas entender por qué

11. /conversations [instancia]
    ESCRIBE: /conversations clinica-bella
    ESCRIBE: /conversations restaurante
    ESCRIBE: /conversations (sin parámetro = TODOS)
    QUÉ PASA: Ver chats activos (de todos o de una instancia)
    PARÁMETROS:
    - instancia: opcional - si no pones, muestra TODOS
    MUESTRA:
    - Nombre cliente/paciente
    - ID del chat (entre backticks)
    - Último mensaje (preview)
    - Hace cuánto fue el último mensaje
    USES: Ver chats de una clínica/restaurante específico

12. /enter [chat_id]
    ESCRIBE: /enter ch_001
    ESCRIBE: /enter patient_maria_123
    ESCRIBE: /enter ch_abc456
    QUÉ PASA: Entrar en una conversación específica
    PARÁMETROS:
    - chat_id: ID del chat (obtenlo de /conversations)
    MUESTRA:
    - Nombre del cliente/paciente
    - ID del chat
    - Histórico de últimos 5 mensajes
    - Sender de cada mensaje
    - Acciones disponibles
    USES: Ver detalles de un cliente específico

════════════════════════════════════════════════════════════════════════════════
                           FLUJOS TÍPICOS DE USO
════════════════════════════════════════════════════════════════════════════════

MONITOREO DIARIO:
  1. /status                    ← ¿Qué está online?
  2. /alerts                    ← ¿Hay problemas?
  3. /conversations             ← ¿Quién habla conmigo?
  4. /help                      ← Si olvidas comandos

CUANDO ALGO FALLA:
  1. /alerts                    ← Qué está roto
  2. /analyze [inst]            ← Entender por qué
  3. /root-cause [evento]       ← Causa específica
  4. /what-happened [inst] 6    ← Qué pasó en últimas 6h
  5. /conversations [inst]      ← Clientes esperando

GESTIONAR CLIENTES:
  1. /conversations             ← Ver todos los chats
  2. "¿Quién está esperando?"   ← Ver pendientes (natural)
  3. /enter [chat_id]           ← Ver detalles del chat

INVESTIGACIÓN PROFUNDA:
  1. /analyze [inst]            ← Análisis completo
  2. /what-happened [inst] 24   ← Últimas 24 horas
  3. /root-cause [evento]       ← Por qué pasó
  4. Preguntas naturales        ← Omni entiende contexto
