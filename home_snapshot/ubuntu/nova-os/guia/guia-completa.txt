Nova OS Guia Completa
=====================

Centro de esta guia
-------------------
- La referencia principal ahora es `guia/inventario-repo.txt`.
- La guia operativa de `Nova + n8n + Gmail` ahora vive en `guia/guia-n8n-gmail-nova.md`.
- La guia de estado real y uso del producto ahora vive en `guia/guia-producto-estado-real.md`.
- Ese archivo contiene el mapa de carpetas y el listado completo de archivos operativos del repositorio, con una descripcion textual por archivo.
- Esta guia completa resume como leer ese inventario y donde estan las piezas mas importantes del sistema.

Lectura recomendada
-------------------
- 1. Leer `README.md` para orientacion general del producto.
- 2. Leer `ARCHITECTURE.md` y `docs/architecture.md` para la vista de sistema.
- 3. Leer `guia/guia-producto-estado-real.md` para saber que esta realmente listo hoy.
- 4. Leer `guia/guia-n8n-gmail-nova.md` si el objetivo es gobernar n8n y correo.
- 5. Ir a `guia/inventario-repo.txt` para localizar cualquier archivo o carpeta concreta.
- 6. Usar `guia/resumen.txt` como ficha rapida para retomar contexto.

Zonas principales del repo
--------------------------
- `nova/`: runtime modular principal de Nova OS.
- `frontend/`: SPA actual React y Vite con branding y dashboard.
- `n8n-nodes-nova/`: nodo comunitario Nova OS para n8n.
- `backend/`: legado, compatibilidad y snapshots empaquetados del backend historico.
- `tests/`: pruebas del runtime, discovery y API.
- `guia/`: documentacion textual local para operacion y mantenimiento.
- `nova.py`: CLI principal, ahora tambien con auth de cuenta web y perfil.

Notas de interpretacion
-----------------------
- El inventario excluye dependencias vendorizadas y salidas compiladas porque no aportan claridad de arquitectura.
- Si una carpeta tiene activos visuales, el inventario los describe como branding, logos o assets publicos.
- Si un modulo fuente no trae docstring explicita, la descripcion se infiere por nombre, carpeta y responsabilidad tecnica.

Archivos que conviene mirar primero
-----------------------------------
- `nova/kernel.py`: orquestador central del runtime.
- `nova/api/server.py`: bootstrap de FastAPI.
- `nova/discovery/scanner.py`: scanner discovery endurecido.
- `nova/discovery/fingerprints.py`: fingerprints estrictos.
- `nova/api/routes/gmail.py`: anti duplicados de Gmail.
- `n8n-nodes-nova/Nova.node.js`: nodo de n8n.
- `frontend/src/pages/dashboard/Discovery.jsx`: experiencia de discovery.
- `frontend/src/pages/dashboard/DiscoveryHelp.jsx`: ayuda operativa de discovery.
- `frontend/src/components/PostLoginOnboarding.jsx`: setup profesional post-login.
- `frontend/src/pages/Settings.jsx`: perfil, personalizacion y reapertura del onboarding.

Referencia central
------------------
- Ver `guia/inventario-repo.txt` para el detalle carpeta por carpeta y archivo por archivo.
- Ver `guia/guia-producto-estado-real.md` para una lectura honesta del estado actual del producto.
- Ver `guia/guia-n8n-gmail-nova.md` para el flujo real de integracion con n8n y Gmail.
