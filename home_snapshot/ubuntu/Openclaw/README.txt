OpenClaw Open Workspace for Melissa pairing scaffolding and internal skill-wrapping notes.

Purpose:
- Provide a safe, local sandbox to design, document, and prototype Melissa-related features (pairing, skill-wrapping, auto-bootstrap) before production integration.
- Serve as a single source of truth for ideas and roadmap, not as the production code itself.

Guiding principles:
- No copying of external proprietary code. Build safe, internal wrappers and adapters.
- Prefer local, self-contained, auditable components.
- Keep channel-specific logic isolated and extendable.
- Document decisions, assumptions, and test plans for future review.

Usage:
- Store design docs, experiments, and planning artifacts here.
- Create new txt/md files for each sub-topic to keep content organized.

Notes:
- This folder is non-production scaffolding. When you verify the designs, we can translate them into code patches in Melissa.
- Do not paste secrets or tokens here.
